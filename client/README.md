# Frontend

Клиентская часть приложения «Custom Hooks».
Описание проекта доступно в [главном README](../README.md).

<!-- Приложение демонстрирует работу c кастомными хуками и взаимодействие с сервером через REST API. -->

## Назначение

Клиент отвечает за отображение результатов запросов к серверу и демонстрацию различных состояний:

- проверка доступности сервера
- отображение успешного ответа
- обработка ошибки запроса
- отображение состояния загрузки

## Интерфейс

- Автоматическая проверка доступности сервера при запуске приложения
- Блоки с результатами запросов - success, error, loading
- Экран ошибки при недоступности сервера
- Loader при первом подключении

## Архитектура

- Используется связка React + TypeScript
- Логика работы с API вынесена в кастомные хуки:
  - `usePing` — проверка доступности сервера перед загрузкой приложения
  - `useJsonFetch` — выполнение HTTP-запросов и обработка состояний
- Каждый запрос вынесен в отдельный компонент `Component`
- Отображение результата централизовано в компоненте `Content`

## Работа приложения

При запуске:

1. Выполняется запрос `/ping` для проверки сервера
2. При успехе отображаются компоненты с запросами:
   - `/success`
   - `/error`
   - `/loading`
3. Каждый компонент отображает одно из состояний:
   - loading
   - error
   - success
4. При ошибке сервера показывается экран ошибки с возможностью повторного запроса
